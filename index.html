To do a cross compilaion, first do
$ sudo apt install gcc-arm-linux-gnueabihf
for arm architecture of raspberry pi
Change directory to a folder that contained a C script
C Script ^
use $ nano to edit the script
Save changes
Compile a test.c to a exucutable file named "test" using
$ arm-linux-gnueabihf-gcc -o test test.c
Transfer test to raspberry pi in a local network by using
$ scp test phumvit@blueberrypi.local:/home/phumvit/cross
The exucutable file test is now in the specific directory named cross on raspberry pi
Remote raspberry pi using
$ ssh phumvit@blueberrypi.local
Change directory to the folder that contained a exucutable file test
Run test by using $ ./test
But in my case, this caused error
Determine the file type by using
$ file ./test
the file is "dynamically linked" when it should be "statically linked"
To fix this, remove the old file from raspberry pi
And also a file from ubuntu
Compile the same script again but force it to be statically linked by using
$ arm-linux-gnueabihf-gcc -static -o test test.c
Transfer test to raspberry pi in a local network by using
$ scp test phumvit@blueberrypi.local:/home/phumvit/cross
Do as previous step and the file is now exucutable
The file is now "statically linked"

$ arduino-cli compile --fqbn arduino:avr:uno /workspace/workspace.ino
